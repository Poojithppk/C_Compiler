## Overview

The Semantic Analysis Phase is the third phase of the NEXUS compiler. It performs deep analysis of the Abstract Syntax Tree (AST) generated by the syntax analysis phase, building a symbol table, checking types, validating scopes, and ensuring semantic correctness of the program.

### Key Responsibilities

1. **Symbol Table Management**
   - Build and maintain a symbol table
   - Manage multiple scope levels (global, function, block)
   - Track symbol declarations and usages

2. **Type Checking**
   - Verify type compatibility in expressions
   - Check function return types
   - Validate assignment types
   - Detect type mismatches

3. **Scope Validation**
   - Verify symbols are declared before use
   - Check for proper scope nesting
   - Detect undefined variable usage
   - Handle scope entry/exit

4. **Semantic Error Detection**
   - Double declaration detection
   - Undefined symbol detection
   - Type mismatch errors
   - Function signature validation

## Components

### 1. `semantic_symbols.py`
Defines the core data structures for semantic analysis:

- **`Symbol`**: Represents a single symbol (variable, function, parameter)
- **`SymbolTable`**: Main symbol table with scope management
- **`Scope`**: Represents a scope level in the program
- **`DataType`**: Enumeration of supported data types
- **`TypeInfo`**: Type information with modifiers (const, pointer, array)
- **`SymbolKind`**: Category of symbols

**Key Classes**:
```python
class SymbolTable:
    - enter_scope()      # Enter new scope level
    - exit_scope()       # Exit current scope
    - declare()          # Declare new symbol
    - lookup()           # Search for symbol in all scopes
    - initialize_symbol() # Mark as initialized
    
class Scope:
    - insert()      # Add symbol to scope
    - lookup()      # Find symbol in this scope only
    - get_all_symbols() # Get all symbols in scope
```

### 2. `semantic.py`
The main semantic analyzer:

- **`SemanticAnalyzer`**: Core analyzer class
  - Visits AST nodes recursively
  - Builds symbol table during traversal
  - Performs type checking
  - Reports semantic errors and warnings

**Key Methods**:
```python
class SemanticAnalyzer:
    - analyze(ast)              # Main analysis method
    - _visit(node)              # Generic AST visitor
    - _visit_VarDeclNode()      # Handle variable declarations
    - _visit_FuncDeclNode()     # Handle function declarations
    - _parse_type()             # Extract type information
    - get_symbol_table()        # Get built symbol table
    - get_analysis_steps()      # Get analysis trace
    - get_statistics()          # Get analysis statistics
```

### 3. `semantic_gui.py`
Visual interface for semantic analysis:

**Features**:
- 📝 **Source Code Editor**: Enter or load code

- 🔍 **Scope Hierarchy**: View scope nesting

```python
from semantic_analysis import SemanticAnalysisGUI
import tkinter as tk

root = tk.Tk()
app = SemanticAnalysisGUI(root)
root.mainloop()
```

### Standalone Use
```python
from semantic_analysis import SemanticAnalyzer, SymbolTable
from syntax_analysis import Parser
from lexical_analysis import VisualLexicalAnalyzer

# Get tokens from lexer
lexer = VisualLexicalAnalyzer()
tokens = lexer.analyze(code)

# Parse to AST
parser = Parser()
ast = parser.parse(tokens)

# Analyze semantics
analyzer = SemanticAnalyzer()
success, errors, warnings = analyzer.analyze(ast)

# Get results
symbol_table = analyzer.get_symbol_table()
stats = analyzer.get_statistics()
```

## Analysis Process

### Step 1: Symbol Table Building
- Traverse AST to find all declarations
- Insert symbols into appropriate scopes
- Check for redeclarations

### Step 2: Variable Analysis
- Verify all variables are declared before use
- Check initialization status
- Track variable lifetime across scopes

### Step 3: Function Analysis
- Analyze function signatures
- Check parameter types
- Validate return types
- Ensure function calls match declarations

### Step 4: Type Checking
- Verify type compatibility in expressions
- Check assignment types
- Validate operator operand types
- Report type mismatches

### Step 5: Error Collection
- Record all semantic errors
- Generate semantic warnings
- Provide error location information

## Symbol Table Structure

```
Global Scope (Level 0)
├── int main() [function]
├── int add(int a, int b) [function]
└── void display(string text) [function]

Function Scope: main (Level 1)
├── int x [variable - initialized]
├── float y [variable - initialized]
└── string message [variable - initialized]

Function Scope: add (Level 1)
├── int a [parameter - initialized]
├── int b [parameter - initialized]
└── int sum [variable - initialized]

Function Scope: display (Level 1)
└── string text [parameter - initialized]
```

## Integration

### With Syntax Analysis
The semantic analyzer takes the AST produced by the parser and performs detailed analysis on it.

### With Next Phase (Code Generation)
After successful semantic analysis, the symbol table and validated AST are used by the intermediate code generation phase.

### Sequential Workflow
```
Lexical Analysis → Syntax Analysis → Semantic Analysis → Code Generation
     (Tokens)          (AST)         (Symbol Table)      (IR Code)
```

## Performance Characteristics

- **Single-pass Analysis**: One traversal of the AST
- **Time Complexity**: O(n) where n = number of nodes in AST
- **Space Complexity**: O(s) where s = number of symbols
- **Scope Depth**: Effectively unlimited (practical: max 50-100 levels)
