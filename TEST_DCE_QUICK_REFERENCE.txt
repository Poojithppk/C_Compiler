📋 6 PHASES OF COMPILATION (What happens to your code):

Phase 1: LEXICAL ANALYSIS
  Input:  Source code text
  Output: Tokens
  Example: "hold x = 10;" → [KEYWORD(hold), ID(x), ASSIGN, NUM(10), SEMI]

Phase 2: SYNTAX ANALYSIS  
  Input:  Tokens
  Output: Abstract Syntax Tree (AST)
  Example: Valid grammar? Yes ✓ → Build AST

Phase 3: SEMANTIC ANALYSIS
  Input:  AST
  Output: Symbol table, type checking
  Example: Is 'x' used before declaration? No ✓

Phase 4: INTERMEDIATE CODE GENERATION
  Input:  Annotated AST
  Output: Three-Address Code (TAC)
  Example: t1 = x + y; → ASSIGN, ADD, etc.

Phase 5: CODE OPTIMIZATION ⭐ (DEAD CODE ELIMINATION HERE)
  Input:  TAC
  Output: Optimized TAC
  Example: Remove unused: t999 = 100;

Phase 6: CODE GENERATION
  Input:  Optimized TAC
  Output: Python, C, or Assembly code
  Example: Convert to real executable code

✓ Dead Code Elimination
  • Removes unused variables
  • Eliminates unreachable code
  • Cleans up dead assignments

✓ Constant Folding
  • 10 + 5 → 15 (computed at compile time)
  • 20 * 2 → 40 (no runtime cost)

✓ Common Subexpression Elimination
  • a + b computed once instead of twice
  • Reuses results

✓ Strength Reduction
  • i * 2 → i << 1 (faster shift vs multiply)
  • i / 4 → i >> 2 (faster than divide)

✓ Peephole Optimization
  • Simplifies instruction sequences
  • Removes redundant operations

✓ Loop Unrolling
  • Reduces loop overhead
  • Increases performance
